# 📘 Go | Day 06 — slice: срезы, `make`, `range`, `copy`

---

## 📖 Теория

### 1. Короткие формы срезов

У `slice` и `array` можно брать под-срезы не только полной формой `a[i:j]`, но и короткими формами:

* `a[:j]` — от начала до `j`, не включая `j`
* `a[i:]` — от `i` до конца
* `a[:]` — весь диапазон

Пример:

```go
nums := []int{10, 20, 30, 40, 50}

fmt.Println(nums[:2]) // [10 20]
fmt.Println(nums[2:]) // [30 40 50]
fmt.Println(nums[:])  // [10 20 30 40 50]
```

Что важно:

* левая граница включается
* правая граница не включается
* `nums[:]` не делает копию данных, а создаёт ещё один `slice` на тот же backing array

---

### 2. `make([]T, len, cap)`

Через `make` можно задать и длину, и вместимость `slice`.

```go
nums := make([]int, 2, 5)

fmt.Println(nums)
fmt.Println(len(nums))
fmt.Println(cap(nums))
```

Результат:

```text
[0 0]
2
5
```

Почему:

* длина `2` — сейчас в `slice` доступны два элемента
* вместимость `5` — в backing array есть место ещё до пяти элементов без обязательного перевыделения памяти

Что важно:

* первые `len` элементов уже существуют и имеют zero value
* `len` всегда меньше или равен `cap`
* если сделать `append`, длина растёт

Пример:

```go
nums := make([]int, 2, 5)
nums[0] = 10
nums[1] = 20

nums = append(nums, 30)

fmt.Println(nums)     // [10 20 30]
fmt.Println(len(nums)) // 3
fmt.Println(cap(nums)) // 5
```

---

### 3. `range` по `slice`

Для обхода `slice` в Go очень часто используют `range`.

```go
nums := []int{10, 20, 30}

for i, v := range nums {
    fmt.Println(i, v)
}
```

Здесь:

* `i` — индекс
* `v` — значение элемента

Если индекс не нужен:

```go
for _, v := range nums {
    fmt.Println(v)
}
```

Если нужно менять элемент по индексу, обычно используют индекс:

```go
for i := range nums {
    nums[i] = nums[i] * 2
}
```

Что важно:

* `range` делает код короче и читаемее для обычного обхода
* `_` используют, когда часть значений не нужна

---

### 4. `append` к `nil slice` и добавление нескольких элементов

`append` умеет работать даже с `nil slice`.

```go
var nums []int

nums = append(nums, 10)
nums = append(nums, 20, 30)

fmt.Println(nums)
```

Результат:

```text
[10 20 30]
```

Что важно:

* `nil slice` — нормальное стартовое значение
* в `append` можно передавать несколько элементов сразу
* результат `append` нужно присваивать обратно

---

### 5. `nil slice` и пустой `slice`

Эти два случая похожи, но не одинаковы.

```go
var a []int
b := []int{}

fmt.Println(a == nil) // true
fmt.Println(b == nil) // false
fmt.Println(len(a), cap(a))
fmt.Println(len(b), cap(b))
```

Что важно:

* у обоих часто `len == 0`
* у обоих часто `cap == 0`
* но только `nil slice` равен `nil`

На практике это бывает важно:

* в проверках `s == nil`
* в сериализации
* в API, где важно различать “нет данных” и “пустой список”

---

### 6. `s[:]` — это не копия

Полный срез:

```go
nums := []int{1, 2, 3}
view := nums[:]

view[0] = 999

fmt.Println(nums) // [999 2 3]
fmt.Println(view) // [999 2 3]
```

Почему:

* `view` и `nums` смотрят на один backing array
* `nums[:]` создаёт ещё одно представление, а не независимую копию

---

### 7. Как сделать независимую копию `slice`

Если нужна именно копия данных, используют `copy`.

```go
src := []int{1, 2, 3}
dst := make([]int, len(src))

copy(dst, src)
dst[0] = 999

fmt.Println(src) // [1 2 3]
fmt.Println(dst) // [999 2 3]
```

Почему:

* `make` создаёт отдельный `slice` со своей памятью
* `copy` переносит значения из `src` в `dst`
* после этого изменение `dst` не влияет на `src`

Что важно:

* `copy` копирует столько элементов, сколько помещается в `dst`
* `copy` не увеличивает длину `dst`

## ⚠️ Типичные ошибки

### Ошибка 1. Думать, что `s[:]` создаёт копию

Нет.

Это ещё один `slice`, который смотрит на тот же backing array.

---

### Ошибка 2. Путать `len` и `cap` в `make([]T, len, cap)`

Например:

```go
nums := make([]int, 2, 5)
```

Это значит:

* уже есть `2` элемента
* вместимость — `5`

Это не значит, что длина тоже `5`.

---

### Ошибка 3. Думать, что `nil slice` и пустой `slice` всегда одно и то же

Они похожи по поведению, но:

* `nil slice == nil`
* пустой `slice` через литерал не равен `nil`

---

### Ошибка 4. Ожидать, что `copy` сам расширит `dst`

`copy` не меняет длину `dst`.

Если `dst` слишком маленький, скопируется только часть данных.

---

### Ошибка 5. Игнорировать `range`, когда нужен обычный обход

Индексный цикл не ошибка, но во многих простых задачах `range` читается лучше.

## 🧠 Полезные приёмы

### 1. Обход через `range`

Когда не нужна сложная индексная логика, это обычно самый читаемый вариант.

```go
sum := 0

for _, v := range nums {
    sum += v
}
```

---

### 2. Сбор результата через пустой `slice` и `append`

Если заранее не знаешь точное количество элементов результата:

```go
result := make([]int, 0)
```

а потом:

```go
result = append(result, value)
```

---

### 3. Независимая копия через `make + copy`

Когда нужно отделить новый `slice` от старого:

```go
dst := make([]int, len(src))
copy(dst, src)
```

Это базовый и очень полезный паттерн.
