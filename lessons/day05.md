# 📘 Go | Day 05 — array / slice

---

## 📖 Теория

### 1. `array`

`array` в Go — это массив фиксированного размера.

Размер массива является частью его типа.

```go
var a [3]int

fmt.Println(a)
```

Что здесь важно:

* `[3]int` и `[5]int` — это разные типы
* размер массива задаётся сразу
* после создания длина массива не меняется

Пример:

```go
nums := [3]int{10, 20, 30}

fmt.Println(nums[0])
fmt.Println(nums[2])
```

---

### 2. Почему `array` используется не так часто

В реальном Go-коде массивы встречаются реже, чем `slice`.

Причина:

* у массива фиксированный размер
* массив копируется целиком при присваивании и передаче в функцию
* для большинства практических задач удобнее использовать `slice`

---

### 3. `slice`

`slice` — это структура-представление над участком массива.

Он не хранит данные “сам по себе” так, как `array`, а ссылается на участок underlying массива.

```go
nums := []int{10, 20, 30}

fmt.Println(nums)
```

Что здесь важно:

* `slice` не фиксирует размер как часть типа
* длина `slice` может меняться
* `slice` чаще всего используется в реальном Go-коде

---

### 4. `array` vs `slice`

Коротко:

* `array` — структура фиксированного размера
* `slice` — представление поверх массива

Главное практическое отличие:

* `array` чаще нужен для понимания устройства Go
* `slice` чаще нужен для написания реального кода

---

### 5. Создание `slice`

Через литерал:

```go
nums := []int{1, 2, 3}
```

Через `make`:

```go
nums := make([]int, 3)

fmt.Println(nums)
```

Через срез массива или другого `slice`:

```go
arr := [5]int{10, 20, 30, 40, 50}
part := arr[1:4]

fmt.Println(part)
```

---

### 6. `nil slice`

У `slice` может быть нулевое значение.

```go
var nums []int

fmt.Println(nums)
fmt.Println(len(nums))
fmt.Println(cap(nums))
```

Что здесь важно:

* `nums == nil` будет `true`
* у такого `slice` длина `0`
* в него можно делать `append`

Пример:

```go
var nums []int
nums = append(nums, 10)

fmt.Println(nums)
```

На практике `nil slice` и пустой `slice` часто ведут себя похоже, но это не всегда одно и то же.

---

### 7. `len` и `cap`

У `slice` есть две важные характеристики:

* `len` — текущая длина
* `cap` — вместимость

```go
nums := []int{10, 20, 30}

fmt.Println(len(nums))
fmt.Println(cap(nums))
```

Как это понимать:

* `len` — сколько элементов сейчас доступно в `slice`
* `cap` — сколько элементов можно вместить до перераспределения памяти

Пример:

```go
arr := [5]int{1, 2, 3, 4, 5}
s := arr[1:3]

fmt.Println(len(s)) // 2
fmt.Println(cap(s)) // 4
```

Почему `cap == 4`:

* `slice` начинается с индекса `1`
* до конца массива остаётся 4 элемента

---

### 8. `append`

`append` добавляет элементы в `slice`.

```go
nums := []int{1, 2, 3}
nums = append(nums, 4)

fmt.Println(nums)
```

Что важно:

* `append` может вернуть новый `slice`
* результат `append` нужно присваивать обратно

Правильно:

```go
nums = append(nums, 4)
```

Неправильно:

```go
append(nums, 4)
```

Во втором случае результат просто потеряется.

---

### 9. Общий backing array

Это одна из самых важных тем.

Если два `slice` смотрят на один и тот же массив, изменение одного может повлиять на другой.

```go
nums := []int{10, 20, 30, 40}
a := nums[1:3]
b := nums[2:4]

a[1] = 999

fmt.Println(a)
fmt.Println(b)
fmt.Println(nums)
```

Что здесь важно:

* `a` и `b` могут разделять одни и те же данные
* изменение через один `slice` иногда видно через другой

Это очень частый источник путаницы.

---

### 10. Когда `append` может создать новый массив

Если текущей вместимости не хватает, `append` создаёт новый underlying массив.

Из-за этого:

* до `append` два `slice` могли делить данные
* после `append` один из них может уже ссылаться на новый массив

Это нужно не заучивать абстрактно, а обязательно проверить руками на примерах.

---

### 11. Копирование `array` и `slice`

Копирование массива:

```go
a := [3]int{1, 2, 3}
b := a

b[0] = 999

fmt.Println(a)
fmt.Println(b)
```

Здесь `a` не изменится, потому что массив копируется целиком.

Копирование `slice`:

```go
a := []int{1, 2, 3}
b := a

b[0] = 999

fmt.Println(a)
fmt.Println(b)
```

Здесь изменение может быть видно в обоих `slice`, потому что они ссылаются на одни данные.

---

## 🔎 Что особенно важно понять

После этого урока ты должен чётко различать:

* `array` — значение фиксированного размера
* `slice` — структура, которая смотрит на массив
* `len` — длина
* `cap` — вместимость
* `append` — может изменить структуру хранения данных

Если эти 5 вещей ясны, дальше тема пойдёт заметно легче.

## ⚠️ Типичные ошибки

### Ошибка 1. Думать, что `array` и `slice` — это почти одно и то же

Нет.

Они связаны, но ведут себя по-разному:

* у массива фиксированный размер
* `slice` гибче
* копирование у них ведёт себя по-разному

---

### Ошибка 2. Забывать присваивать результат `append`

```go
nums := []int{1, 2, 3}
append(nums, 4)
```

Так делать нельзя, потому что новый `slice` не сохраняется.

---

### Ошибка 3. Не понимать, почему изменение одного `slice` меняет другой

Причина часто одна:

* оба `slice` смотрят на один backing array

---

### Ошибка 4. Путать `len` и `cap`

* `len` — сколько элементов сейчас в `slice`
* `cap` — сколько он может вместить до перераспределения

---

### Ошибка 5. Передавать большой `array`, думая как о `slice`

Массив — это значение.

Если передать его в функцию, он будет копироваться, если не использовать указатель.

## 🧠 Полезные паттерны

### 1. Использовать `slice` по умолчанию, если не нужен фиксированный размер

Для большинства прикладных задач это правильный выбор.

---

### 2. Проверять руками, делят ли `slice` один backing array

На этой теме очень полезны маленькие эксперименты:

* создать базовый `slice`
* взять из него 2 под-среза
* изменить один
* посмотреть, что произошло со вторым

---

### 3. Замечать, где копия данных, а где общее представление

Это важнее, чем просто знать синтаксис.

На собеседовании и в реальной работе часто важен не ответ “как написать”, а понимание “что реально хранится в памяти”.
