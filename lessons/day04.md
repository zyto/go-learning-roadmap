# 📘 Go | Day 04 — break / continue / вложенные циклы

---

## 📖 Теория

### 1. `break`

`break` полностью завершает цикл, в котором он находится.

```go
for i := 1; i <= 10; i++ {
    if i == 6 {
        break
    }

    fmt.Println(i)
}
```

Что произойдёт:

* цикл остановится при `i == 6`
* в вывод попадут числа от `1` до `5`

---

### 2. `continue`

`continue` не завершает цикл. Он пропускает оставшийся код текущей итерации и переходит к следующей.

```go
for i := 1; i <= 5; i++ {
    if i == 3 {
        continue
    }

    fmt.Println(i)
}
```

Что произойдёт:

* число `3` не будет выведено
* цикл продолжит работу дальше

---

### 3. Разница между `break` и `continue`

* `break` — остановить цикл полностью
* `continue` — пропустить только текущую итерацию

Короткая модель в голове:

* `break` = "дальше крутиться не нужно"
* `continue` = "этот элемент пропускаем, смотрим следующий"

---

### 4. Вложенные циклы

Вложенный цикл — это цикл внутри другого цикла.

```go
for i := 1; i <= 3; i++ {
    for j := 1; j <= 2; j++ {
        fmt.Printf("i=%d j=%d\n", i, j)
    }
}
```

Как это читать:

* внешний цикл управляет "большими шагами"
* внутренний цикл полностью выполняется на каждой итерации внешнего

Если внешний цикл выполняется `3` раза, а внутренний `2` раза, то внутреннее действие выполнится `6` раз.

---

### 5. Где часто применяют вложенные циклы

* перебор пар значений
* работа с таблицами
* вывод фигур символами
* поиск совпадений между двумя наборами данных

---

## 🔎 Важный нюанс: на какой цикл влияют `break` и `continue`

Без меток (`labels`) и `break`, и `continue` работают только с ближайшим циклом.

```go
for i := 1; i <= 3; i++ {
    for j := 1; j <= 3; j++ {
        if j == 2 {
            break
        }

        fmt.Printf("i=%d j=%d\n", i, j)
    }
}
```

Что здесь важно:

* `break` завершит только внутренний цикл `j`
* внешний цикл `i` продолжит выполняться

---

## 🏷 Метки (`labels`) для внешнего цикла

Иногда из внутреннего цикла нужно повлиять не на ближайший, а на внешний цикл. Для этого в Go есть метки.

```go
Outer:
for i := 1; i <= 3; i++ {
    for j := 1; j <= 3; j++ {
        if i == 2 && j == 2 {
            break Outer
        }
    }
}
```

Что здесь важно:

* `Outer` — не специальное слово, а имя метки
* имя можно выбрать другое: `Rows`, `MainLoop`, `SearchLoop`
* `break Outer` завершит именно внешний цикл

---

### `continue` с меткой

```go
Outer:
for i := 1; i <= 3; i++ {
    for j := 1; j <= 3; j++ {
        if j == 2 {
            continue Outer
        }

        fmt.Printf("i=%d j=%d\n", i, j)
    }
}
```

Что произойдёт:

* как только `j == 2`, управление перейдёт к следующей итерации внешнего цикла
* внутренний цикл не продолжится до `j == 3`

## ⚠️ Типичные ошибки

### Ошибка 1. Путать `break` и `continue`

```go
if x == 10 {
    continue
}
```

Этот код не завершит цикл. Он только пропустит текущую итерацию.

---

### Ошибка 2. Думать, что `break` во внутреннем цикле остановит все циклы

Без метки это не так. Остановится только ближайший цикл.

---

### Ошибка 3. Не понимать, сколько раз реально выполняется внутренний цикл

Если:

* внешний цикл идёт `4` раза
* внутренний цикл идёт `3` раза

то тело внутреннего цикла выполнится `12` раз.

---

### Ошибка 4. Делать слишком сложную вложенность без необходимости

Если код становится тяжёлым для чтения:

* проверь, можно ли упростить условие
* можно ли вынести часть логики в отдельную функцию
* не лучше ли использовать `continue`, чтобы уменьшить вложенность

## 🧠 Полезные паттерны

### 1. Фильтрация через `continue`

Это частый и удобный стиль: сначала отбрасываем неподходящие значения, потом работаем с подходящими.

```go
for i := 1; i <= 10; i++ {
    if i%2 == 0 {
        continue
    }

    fmt.Println(i)
}
```

Плюс такого подхода:

* меньше лишней вложенности
* код читается проще

---

### 2. Ранний выход через `break`

Когда результат уже найден, нет смысла продолжать цикл.

```go
found := false

for i := 1; i <= 10; i++ {
    if i == 7 {
        found = true
        break
    }
}

fmt.Println(found)
```

---

### 3. Вложенные циклы для построения формы

```go
for row := 1; row <= 3; row++ {
    for col := 1; col <= 5; col++ {
        fmt.Print("*")
    }
    fmt.Println()
}
```

Так обычно строят:

* прямоугольники
* таблицы
* сетки
