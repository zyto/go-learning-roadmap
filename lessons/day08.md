# 📘 Go | Day 08 — `string`, `byte`, `rune`, `UTF-8`

---

## 📖 Теория

### 1. Что такое `string`

`string` в Go — это строка.

На практике строка хранит последовательность байтов.

Пример:

```go
s := "hello"

fmt.Println(s)
fmt.Println(len(s))
```

Результат:

```text
hello
5
```

Почему:

* строка содержит 5 латинских символов
* в ASCII каждый такой символ занимает 1 байт
* поэтому `len(s)` равно `5`

Что здесь важно:

* строка в Go тесно связана с байтами
* для ASCII часто кажется, что “байт” и “символ” — это одно и то же
* но для Unicode-строк это уже не так

---

### 2. `len(string)` считает байты, а не символы

Это один из самых важных моментов темы.

Пример:

```go
s := "привет"

fmt.Println(s)
fmt.Println(len(s))
```

Результат:

```text
привет
12
```

Почему:

* в слове `привет` 6 букв
* в UTF-8 каждая из этих букв занимает 2 байта
* поэтому `len(s)` возвращает `12`

Главный вывод:

* `len(string)` — это длина в байтах
* количество “видимых букв” и количество байтов — не одно и то же

---

### 3. Индексация строки возвращает `byte`

Когда мы обращаемся к строке по индексу, мы получаем не “символ”, а один байт.

Пример:

```go
s := "go"

fmt.Println(s[0])
fmt.Println(string(s[0]))
```

Результат:

```text
103
g
```

Почему:

* `s[0]` — это первый байт строки
* байт буквы `g` в ASCII имеет значение `103`
* `string(s[0])` превращает этот байт обратно в строку из одного символа

Для ASCII это выглядит удобно.

Но с Unicode-строкой картина меняется.

Пример:

```go
s := "пр"

fmt.Println(len(s))
fmt.Println(s[0])
fmt.Println(s[1])
```

Результат:

```text
4
208
191
```

Почему:

* строка `пр` состоит из 2 букв
* в UTF-8 это 4 байта
* `s[0]` и `s[1]` — это только первые два байта, а не две отдельные буквы

Что здесь важно:

* индексировать строку можно
* но результат — это `byte`
* для Unicode-строк индекс по строке не равен индексу по “символам”

---

### 4. Что такое `byte`

`byte` в Go — это псевдоним для `uint8`.

То есть `byte` — это просто один байт данных.

Пример:

```go
s := "go"
b := []byte(s)

fmt.Println(b)
fmt.Println(len(b))
```

Результат:

```text
[103 111]
2
```

Почему:

* строка `"go"` состоит из двух ASCII-байтов
* при преобразовании к `[]byte` мы получаем эти байты явно

Где это полезно:

* работа с файлами
* сеть и протоколы
* JSON, HTTP, логирование
* ситуации, где важны именно байты, а не текстовые символы

---

### 5. Что такое `rune`

`rune` в Go — это псевдоним для `int32`.

На практике `rune` используют для представления одного Unicode code point.

Проще всего думать так:

* `byte` — один байт
* `rune` — один текстовый символ на уровне Unicode-кода

Пример:

```go
s := "привет"
runes := []rune(s)

fmt.Println(runes)
fmt.Println(len(runes))
```

Результат:

```text
[1087 1088 1080 1074 1077 1090]
6
```

Почему:

* `[]rune(s)` разбирает строку по Unicode-символам
* в слове `привет` 6 букв
* поэтому длина `[]rune` равна `6`

Что здесь важно:

* `[]byte(s)` и `[]rune(s)` — это не одно и то же
* байтов у строки может быть больше, чем рун

---

### 6. `range` по строке работает с `rune`

Цикл `range`, который уже использовался со `slice` и `map`, умеет корректно обходить строку по рунам.

Пример:

```go
s := "кот"

for i, r := range s {
    fmt.Println(i, string(r))
}
```

Результат:

```text
0 к
2 о
4 т
```

Почему:

* `r` — это очередная `rune`
* `i` — это не номер символа, а позиция в байтах
* каждая кириллическая буква здесь занимает 2 байта

Это очень важный момент:

* `range` по строке даёт руны
* индекс в `range` по строке — байтовый

---

### 7. `[]byte` и `[]rune` дают разный взгляд на одну и ту же строку

Пример:

```go
s := "кот"
bytes := []byte(s)
runes := []rune(s)

fmt.Println(len(bytes))
fmt.Println(len(runes))
```

Результат:

```text
6
3
```

Почему:

* в слове `кот` 3 буквы
* каждая буква занимает 2 байта в UTF-8
* поэтому байтов 6, а рун 3

Как это понимать:

* `[]byte` нужен, когда важны сырые данные
* `[]rune` нужен, когда важны текстовые символы

---

### 8. Строки в Go неизменяемые

Строку нельзя поменять по индексу напрямую.

Такой код не скомпилируется:

```go
s := "go"

// s[0] = 'n'
```

Причина:

* `string` в Go immutable
* можно читать строку, брать её длину, обходить, преобразовывать
* но нельзя просто заменить байт или руну внутри исходной строки

Если строку нужно изменить, обычно делают так:

* преобразуют её в `[]rune` или `[]byte`
* меняют данные
* собирают новую строку обратно

---

### 9. Как безопасно менять строку

Если нужно изменить именно символы, а не байты, лучше работать через `[]rune`.

Пример:

```go
s := "кот"
runes := []rune(s)

runes[0] = 'т'

fmt.Println(string(runes))
```

Результат:

```text
тот
```

Почему:

* строка преобразована в `[]rune`
* запись `'т'` — это `rune`-литерал, то есть один символ в одинарных кавычках
* теперь можно менять отдельные символы по индексу
* `string(runes)` собирает новую строку обратно

---

### 10. Что такое `UTF-8`

`UTF-8` — это способ кодирования Unicode-символов в байты.

Главная идея:

* разные символы могут занимать разное количество байтов

Пример:

```go
fmt.Println(len("A"))
fmt.Println(len("Ж"))
fmt.Println(len("🙂"))
```

Результат:

```text
1
2
4
```

Почему:

* латинская `A` занимает 1 байт
* кириллическая `Ж` занимает 2 байта
* emoji `🙂` занимает 4 байта

Отсюда и возникает большинство “странностей” со строками:

* длина в байтах не совпадает с количеством символов
* индекс по байтам не равен индексу по буквам

---

### 11. Реверс строки нужно делать через `rune`

Если разворачивать Unicode-строку как набор байтов, можно сломать текст.

Безопасный вариант — сначала превратить строку в `[]rune`.

Пример:

```go
s := "привет"
runes := []rune(s)
result := make([]rune, 0, len(runes))

for i := len(runes) - 1; i >= 0; i-- {
    result = append(result, runes[i])
}

fmt.Println(string(result))
```

Результат:

```text
тевирп
```

Почему:

* мы работаем не с байтами, а с рунами
* каждая буква остаётся целой
* итоговая строка корректно собирается обратно

---

### 12. Фильтрация символов тоже обычно делается через `rune`

Если нужно убрать из строки некоторые символы, удобно пройтись по ней через `range` и собрать новый `[]rune`.

Пример:

```go
s := "go-lang-go"
result := make([]rune, 0)

for _, r := range s {
    if r != '-' {
        result = append(result, r)
    }
}

fmt.Println(string(result))
```

Результат:

```text
golanggo
```

Почему:

* `range` отдаёт строку по рунам
* мы пропускаем только `-`
* остальные символы собираем в новый результат

Этот паттерн очень полезен для:

* очистки пользовательского ввода
* удаления пробелов, разделителей, служебных символов
* подготовки текста перед дальнейшей обработкой

---

## 🔎 Что особенно важно понять

После этого урока ты должен чётко различать:

* `string` — это строка, которая хранит байты
* `len(string)` считает байты
* `byte` — это один байт данных
* `rune` — это Unicode-символ на уровне code point
* `range` по строке идёт по `rune`, а индекс отдаёт в байтах
* для изменения или разворота Unicode-строки обычно нужен `[]rune`
* строки в Go неизменяемые
* `UTF-8` объясняет, почему один символ может занимать 1, 2 или 4 байта

Если эти вещи не путать, работа со строками в Go станет гораздо спокойнее.

## ⚠️ Типичные ошибки

### Ошибка 1. Думать, что `len(s)` — это количество букв

Для ASCII это часто выглядит правдой.

Но для Unicode это не так.

Пример:

```go
fmt.Println(len("привет"))
```

Результат:

```text
12
```

Хотя букв здесь `6`.

---

### Ошибка 2. Думать, что `s[i]` возвращает символ

Нет.

`s[i]` возвращает один байт.

Это особенно заметно на кириллице, emoji и других Unicode-символах.

---

### Ошибка 3. Пытаться обрабатывать Unicode-строку как `[]byte`, когда нужны именно символы

Если задача про:

* реверс строки
* замену букв
* фильтрацию символов
* подсчёт символов

то чаще нужен `[]rune`, а не `[]byte`.

---

### Ошибка 4. Пытаться менять строку по индексу

Так делать нельзя, потому что `string` неизменяемый.

Нужен путь:

* `string -> []rune`
* изменить данные
* `[]rune -> string`

---

### Ошибка 5. Путать байтовый индекс и номер символа

В цикле:

```go
for i, r := range s
```

* `r` — это руна
* `i` — позиция в байтах

Это не одно и то же.

---

## 🧠 Полезные паттерны

### 1. Использовать `range` для чтения строки

Если тебе нужно пройтись по тексту посимвольно, базовый и безопасный путь:

```go
for _, r := range s
```

Это почти всегда лучше, чем ручная работа по индексам байтов, если строка может содержать Unicode.

---

### 2. Использовать `[]rune` для трансформации строки

Если строку нужно:

* изменить
* развернуть
* отфильтровать

то удобный путь:

* `runes := []rune(s)`
* работа с `runes`
* `string(runes)`

---

### 3. Использовать `[]byte`, когда важны именно байты

Это уместно, когда работа идёт не с “текстом как набором букв”, а с данными:

* чтение и запись
* сетевые пакеты
* сериализация
* побайтовая обработка

---

### 4. Использовать `map[rune]int` для подсчёта символов

Тема `map` уже знакома с прошлого урока, и здесь она хорошо сочетается со строками.

Пример идеи:

```go
counts := make(map[rune]int)

for _, r := range s {
    counts[r] = counts[r] + 1
}
```

Это полезно для:

* частотного анализа
* поиска повторов
* простых задач на текстовую обработку

## 🧠 Сравнение с практикой backend

Где это реально пригодится:

* валидация имён, логинов, тегов и других пользовательских строк
* очистка входных данных перед сохранением
* работа с JSON, HTTP и файлами, где часто важны именно байты
* фильтрация и нормализация текста
* безопасная обработка Unicode-строк в API и логах

Очень частая практическая ошибка в backend:

* код отлично работает на ASCII
* а потом начинает вести себя странно на кириллице или emoji

Обычно причина одна:

* разработчик перепутал байты и символы
