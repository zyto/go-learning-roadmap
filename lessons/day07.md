# 📘 Go | Day 07 — `map`: словари, `ok`-идиома, частотный словарь, set

---

## 📖 Теория

### 1. Что такое `map`

`map` в Go — это структура данных вида “ключ -> значение”.

Она нужна, когда по какому-то ключу нужно быстро находить связанное значение.

Пример:

```go
ages := map[string]int{
    "Alice": 30,
    "Bob":   25,
}

fmt.Println(ages)
fmt.Println(ages["Alice"])
```

Результат:

```text
map[Alice:30 Bob:25]
30
```

Что здесь важно:

* `string` — тип ключа
* `int` — тип значения
* по ключу `"Alice"` можно получить значение `30`

На практике `map` очень часто используют для:

* быстрого поиска по ID или имени
* подсчёта количества повторений
* проверки “видели ли мы уже это значение”
* группировки данных

---

### 2. Как создать `map`

Есть два основных способа.

Через литерал:

```go
scores := map[string]int{
    "math":    5,
    "english": 4,
}
```

Через `make`:

```go
scores := make(map[string]int)

scores["math"] = 5
scores["english"] = 4

fmt.Println(scores)
```

Что важно:

* литерал удобен, когда данные уже известны сразу
* `make(map[K]V)` удобен, когда `map` сначала пустой, а потом будет заполняться
* чаще всего в начале обучения достаточно ключей типа `string` или `int`

---

### 3. Чтение и запись в `map`

Запись:

```go
prices := make(map[string]int)

prices["apple"] = 120
prices["banana"] = 90
```

Чтение:

```go
fmt.Println(prices["apple"])
fmt.Println(prices["banana"])
```

Результат:

```text
120
90
```

Что важно:

* `m[key] = value` — запись
* `m[key]` — чтение
* если ключ уже есть, новое присваивание просто перезапишет старое значение

Пример:

```go
prices["apple"] = 150

fmt.Println(prices["apple"])
```

Результат:

```text
150
```

---

### 4. Zero value при чтении из `map`

Если ключа нет, чтение из `map` не падает с ошибкой.

Go просто возвращает zero value типа значения.

Пример:

```go
prices := map[string]int{
    "apple": 120,
}

fmt.Println(prices["apple"])
fmt.Println(prices["orange"])
```

Результат:

```text
120
0
```

Почему:

* ключ `"apple"` есть, поэтому возвращается его значение
* ключа `"orange"` нет, поэтому для типа `int` возвращается zero value, то есть `0`

Это очень важный момент.

Для разных типов zero value будет разным:

* для `int` — `0`
* для `string` — `""`
* для `bool` — `false`
* для `[]int` — `nil`

---

### 5. Почему нужна `ok`-идиома

Проблема в том, что по одному только `m[key]` не всегда понятно:

* ключа нет
* или ключ есть, но в нём лежит zero value

Пример:

```go
stock := map[string]int{
    "apple": 0,
}

fmt.Println(stock["apple"])
fmt.Println(stock["banana"])
```

Результат:

```text
0
0
```

По этому выводу нельзя понять:

* `"apple"` есть в `map`, просто значение равно `0`
* `"banana"` нет в `map` вообще

Для этого и используют `ok`-идиому:

```go
stock := map[string]int{
    "apple": 0,
}

value, ok := stock["apple"]
fmt.Println(value, ok)

value, ok = stock["banana"]
fmt.Println(value, ok)
```

Результат:

```text
0 true
0 false
```

Что важно:

* `value` — это значение
* `ok == true` значит ключ найден
* `ok == false` значит ключа нет

Именно так обычно и проверяют наличие ключа в `map`.

---

### 6. `nil map`

У `map` есть нулевое значение.

```go
var scores map[string]int

fmt.Println(scores == nil)
fmt.Println(scores["math"])
```

Результат:

```text
true
0
```

Что здесь важно:

* `scores` пока равен `nil`
* читать из `nil map` можно
* при чтении снова возвращается zero value

Но записывать в `nil map` нельзя:

```go
var scores map[string]int
scores["math"] = 5
```

Это приведёт к panic во время выполнения.

Поэтому:

* если `map` будет заполняться, обычно используют `make`
* `var m map[K]V` без `make` годится только если ты точно понимаешь, зачем это делаешь

---

### 7. `delete`

Удаление ключа делается через `delete`.

```go
users := map[string]int{
    "alice": 1,
    "bob":   2,
    "carol": 3,
}

delete(users, "bob")

fmt.Println(users)
```

Что важно:

* `delete(m, key)` удаляет ключ из `map`
* если ключа нет, `delete` не падает и ничего страшного не происходит

Пример:

```go
delete(users, "nobody")
```

Это нормальная операция.

---

### 8. `range` по `map`

`range`, который уже использовался со `slice`, работает и с `map`.

```go
scores := map[string]int{
    "math":    5,
    "english": 4,
    "history": 5,
}

for subject, score := range scores {
    fmt.Println(subject, score)
}
```

Что важно:

* в цикле получаем ключ и значение
* порядок обхода `map` не гарантирован

Это очень важный практический момент:

* нельзя рассчитывать, что `map` будет печататься или обходиться в одном и том же порядке
* `map` — это не упорядоченный список

---

### 9. Частотный словарь

Один из самых полезных паттернов с `map` — считать, сколько раз встретилось каждое значение.

Пример:

```go
words := []string{"go", "php", "go", "sql", "go", "php"}
counts := make(map[string]int)

for _, word := range words {
    counts[word] = counts[word] + 1
}

fmt.Println(counts)
```

Как это работает:

* сначала `counts` пустой
* если слова ещё не было, `counts[word]` даст `0`
* потом мы прибавляем `1`
* если слово уже встречалось, счётчик увеличится ещё раз

Это как раз пример, где zero value у `map` очень удобен.

---

### 10. Set-паттерн через `map`

В Go часто делают “множество” через `map`, чтобы быстро проверять, встречалось значение или нет.

На текущем этапе самый простой вариант:

```go
seen := make(map[string]bool)
```

Пример:

```go
names := []string{"alice", "bob", "alice", "carol", "bob"}
seen := make(map[string]bool)
result := make([]string, 0)

for _, name := range names {
    if !seen[name] {
        seen[name] = true
        result = append(result, name)
    }
}

fmt.Println(result)
```

Результат:

```text
[alice bob carol]
```

Что здесь важно:

* `seen[name]` вернёт `false`, если имени ещё не было
* после этого мы помечаем имя как уже встреченное
* такой паттерн очень удобен для дедупликации

---

### 11. Индексация данных через `map`

Ещё один частый сценарий — построить быстрый индекс по ключу.

Пример:

```go
ids := []int{101, 205, 330}
names := []string{"Alice", "Bob", "Carol"}

index := make(map[int]string)

for i := range ids {
    index[ids[i]] = names[i]
}

fmt.Println(index[205])
```

Почему это полезно:

* вместо перебора всего `slice` можно сразу обратиться по ключу
* этот паттерн очень часто встречается в backend-коде

---

### 12. Базовый `group by`

`map` удобно использовать и для группировки данных.

Пример:

```go
departments := []string{"backend", "frontend", "backend", "qa"}
names := []string{"Ira", "Oleg", "Anton", "Lena"}

groups := make(map[string][]string)

for i := range departments {
    dept := departments[i]
    name := names[i]

    groups[dept] = append(groups[dept], name)
}

fmt.Println(groups)
```

Что здесь происходит:

* ключ — это название группы, например отдел
* значение — `slice`, в который собираются элементы этой группы
* `append(groups[dept], name)` работает, даже если `groups[dept]` пока `nil`

Это очень полезный прикладной паттерн:

* `map[K][]V`

---

## 🔎 Что особенно важно понять

После этого урока ты должен чётко различать:

* `map` — это структура “ключ -> значение”
* чтение отсутствующего ключа возвращает zero value
* `ok`-идиома нужна, когда важно отличать “ключа нет” от “значение равно zero value”
* `nil map` можно читать, но в неё нельзя писать
* `delete` безопасно удаляет ключ
* `range` по `map` не гарантирует порядок
* `map` особенно полезен для частотного словаря, set-паттерна, индексации и группировки

## ⚠️ Типичные ошибки

### Ошибка 1. Думать, что `m[key] == 0` всегда значит “ключа нет”

Нет.

Это может значить два разных случая:

* ключа нет
* ключ есть, но значение реально равно `0`

Если это важно, нужен вариант:

```go
value, ok := m[key]
```

---

### Ошибка 2. Писать в `nil map`

Пример:

```go
var m map[string]int
m["go"] = 1
```

Так делать нельзя.

Для записи нужна инициализация:

```go
m := make(map[string]int)
```

---

### Ошибка 3. Ожидать фиксированный порядок элементов в `map`

`map` не хранит данные как упорядоченный список.

Если сделать:

```go
for k, v := range m {
    fmt.Println(k, v)
}
```

нельзя рассчитывать на один и тот же порядок обхода.

---

### Ошибка 4. Использовать `map`, когда нужен обычный список

Если тебе важен порядок элементов и нужен просто последовательный набор значений, чаще подходит `slice`.

`map` хорош, когда нужен доступ по ключу.

---

### Ошибка 5. Не замечать, что `map[K][]V` удобно для группировки

Многие сначала пытаются делать группировку вручную через вложенные циклы.

Но если задача именно “собрать элементы по ключу”, часто естественнее использовать:

* `map[K][]V`

---

## 🧠 Полезные паттерны

### 1. Частотный словарь

Когда нужно посчитать, сколько раз встретилось каждое значение:

```go
counts[value] = counts[value] + 1
```

Это базовый паттерн для:

* статистики
* поиска дубликатов
* агрегации данных

---

### 2. Set-паттерн для дедупликации

Когда нужно запомнить, что значение уже встречалось:

```go
seen := make(map[string]bool)
```

А дальше:

```go
if !seen[value] {
    seen[value] = true
}
```

Это удобно для:

* удаления повторов
* быстрой проверки наличия
* защиты от повторной обработки

---

### 3. Индекс по ключу

Когда данные из `slice` нужно потом быстро находить по ID или имени:

```go
index[id] = value
```

Это частый практический мостик между “просто список данных” и “быстрый lookup”.

---

### 4. `group by` через `map[K][]V`

Когда нужно собирать данные по категориям:

```go
groups[key] = append(groups[key], value)
```

Это удобно для:

* группировки по статусу
* группировки по типу
* подготовки данных для ответа API или отчёта

---

## 🧠 Сравнение с практикой backend

Где это реально пригодится:

* индексирование данных по ID
* группировка записей по статусу или категории
* подсчёт количества ошибок, тегов, типов событий
* дедупликация входных данных
* подготовка промежуточных структур перед JSON-ответом или записью в БД

Очень часто `map` в backend — это не “академическая структура данных”, а обычный рабочий инструмент для:

* lookup
* aggregation
* grouping
* deduplication
