# Профиль ученика

## Сильные стороны

* Большой опыт коммерческой разработки
* Хорошая база SQL
* Понимание бизнес-логики
* Есть backend-опыт на PHP

## Ограничения и риски

* Go изучается с нуля
* Пока слабое понимание памяти и runtime-поведения Go
* Нет практики конкурентности в Go
* Возможен перенос неидиоматичных привычек из 1С / PHP

## Вывод для программы

Обучение должно быть ориентировано не на базовое “научиться программировать”, а на:

* быстрое освоение синтаксиса и идиом Go
* переход к backend-практике как можно раньше
* регулярную проверку понимания через review и вопросы
* акцент на объяснение решений и trade-offs

# Правила обучения

## Основной принцип

> 70% практика / 30% теория

## Рабочий цикл

1. Определяется тема и учебная цель
2. Даётся короткая теория и набор задач
3. Пользователь пишет код сам
4. Код проходит ревью
5. Закрываются ошибки, пробелы и вопросы
6. Тема считается завершённой только после проверки понимания

## Ограничения

* Не решать задачи полностью без попытки пользователя
* Не перепрыгивать через фундаментальные темы без явного запроса
* Не считать тему завершённой без практической проверки

## Роль Codex

Codex используется как:

* reviewer
* debugger
* mentor
* explainer
* mock interviewer

При review:

* сначала указывать ошибки, риски и пробелы
* потом объяснять причины и направление исправления

# Программа обучения

Программа делится на 5 фаз:

1. `Phase 1 — Core Go Foundations`
2. `Phase 2 — Backend Basics in Go`
3. `Phase 3 — Data Layer and Persistence`
4. `Phase 4 — Concurrency and Production Patterns`
5. `Phase 5 — Architecture, Testing, Interview Readiness`

## Phase 1 — Core Go Foundations

### Цель

Научиться уверенно писать небольшие и средние программы на Go, понимать базовые структуры данных, функции, методы, ошибки и ключевые идиомы языка.

### Темы

* синтаксис и управление потоком:
  `if / else`, `switch`, `for`, `break`, `continue`, вложенные циклы
* структуры данных:
  `array`, `slice`, `map`, `string`, `byte`, `rune`
* функции и организация кода:
  параметры, return values, `defer`, scope
* типы и поведение:
  `struct`, методы, pointer/value receiver, базовые интерфейсы
* надежность:
  `error`, wrapping, `errors.Is`, `errors.As`
* устройство кода:
  `package`, экспорт, stdlib-first mindset

### Паттерны и антипаттерны

* паттерны:
  early return, guard clauses, small functions, zero value friendly design, composition over inheritance
* антипаттерны:
  лишняя вложенность, premature abstraction, интерфейсы "на будущее", `panic` вместо `error`, размытые имена вроде `utils` / `helpers`

### Алгоритмический минимум

* проход по массиву и `slice`
* суммирование и поиск минимума / максимума
* фильтрация значений
* частотный словарь через `map`
* поиск дубликатов
* реверс строки с учетом `rune`
* двумерный проход через вложенные циклы

### Результат фазы

* 40-50 маленьких упражнений
* 1 консольный мини-проект
* 2 mini mock interview

## Phase 2 — Backend Basics in Go

### Цель

Научиться писать простой, но корректный backend на Go без магии фреймворков.

### Темы

* `net/http`
* handlers
* routing
* query params / path params
* HTTP status codes
* `encoding/json`
* request / response structure
* middleware
* logging
* config
* env vars
* graceful shutdown
* `context` в request lifecycle

### Паттерны и антипаттерны

* паттерны:
  handler -> service split, explicit error handling, middleware chain, dependency injection без переусложнения, явная валидация входных данных
* антипаттерны:
  жирные handlers, бизнес-логика в HTTP-слое, магические JSON-ответы, размазанная валидация, скрытые side effects

### Алгоритмический минимум

* фильтрация коллекций
* дедупликация записей
* пагинация в памяти
* сортировка списка структур
* группировка данных для ответа API

### Результат фазы

* 1 mini CRUD API без БД
* разбор структуры handler -> service
* review backend-кода

## Phase 3 — Data Layer and Persistence

### Цель

Научиться работать с PostgreSQL из Go и строить базовый persistence layer.

### Темы

* `database/sql`
* PostgreSQL
* CRUD
* фильтрация
* пагинация
* транзакции
* индексы
* migrations
* repository pattern

### Паттерны и антипаттерны

* паттерны:
  repository pattern, transaction boundary, явные контракты между service и repository, подготовка структур данных до передачи в storage
* антипаттерны:
  SQL в случайных местах кода, N+1 мышление, неявные транзакции, смешивание бизнес-логики и SQL-деталей

### Алгоритмический минимум

* grouping и aggregation
* merge / transform нескольких наборов данных
* индексирование данных через `map`
* подготовка lookup-структур для ускорения обработки

### Результат фазы

* CRUD API с PostgreSQL
* минимум один сценарий с транзакцией
* 2-3 SQL-запроса с фильтрацией и пагинацией

## Phase 4 — Concurrency and Production Patterns

### Цель

Понять конкурентность в Go на уровне, достаточном для уверенной backend-практики.

### Темы

* goroutines
* channels
* buffered / unbuffered channels
* `select`
* mutex
* race condition
* worker pool
* `context` cancellation

### Паттерны и антипаттерны

* паттерны:
  worker pool, bounded concurrency, cancellation через `context`, fan-out / fan-in на базовом уровне
* антипаттерны:
  goroutine leak, shared mutable state без синхронизации, каналы "на всё подряд", игнорирование race conditions

### Алгоритмический минимум

* batching задач
* pipeline-обработка
* fan-out обработки набора данных
* координация результатов из нескольких goroutines

### Результат фазы

* 10-15 упражнений
* 1 прикладной модуль с параллельной обработкой
* разбор типовых race conditions

## Phase 5 — Architecture, Testing, Interview Readiness

### Цель

Собрать знания в production-like подход и подготовиться к собеседованию на middle Go backend.

### Темы

* слои
* dependency injection
* clean architecture basics
* testability
* unit tests
* table-driven tests
* integration tests
* границы unit / integration tests
* code review simulation
* backend interview questions

### Паттерны и антипаттерны

* паттерны:
  layered architecture, dependency injection without overengineering, table-driven tests, explicit boundaries between handler / service / repository
* антипаттерны:
  архитектура ради архитектуры, избыточные интерфейсы на CRUD, mocks everywhere, premature generalization

### Алгоритмический минимум

* 10-15 коротких interview-style задач на уже изученные структуры данных
* упражнения на чтение и разбор чужого кода
* задачи на поиск ошибок в логике, а не только на написание решения с нуля

### Результат фазы

* 2 проекта в портфолио
* серия mock interview
* финальный список слабых мест и план добивки

# Наработки для следующего урока

## Day 05 — array / slice

### Теория

* `array`
* `slice`
* отличие `slice` от `array`
* длина `len`
* вместимость `cap`
* базовые операции со `slice`
* `append`
* общий backing array на базовом уровне

### На что обратить внимание

* почему `slice` используется чаще `array`
* что именно хранит `slice`
* чем отличается копирование `array` от копирования `slice`
* когда `append` может создать новый backing array
* почему изменение одного `slice` иногда влияет на другой

### Практика

1. объявить `array` из нескольких чисел и вывести его элементы
2. создать `slice`, изменить один элемент и вывести результат
3. сравнить `len` и `cap` на нескольких примерах
4. использовать `append` для добавления элементов
5. проверить, как ведут себя два `slice`, если они ссылаются на одни данные
